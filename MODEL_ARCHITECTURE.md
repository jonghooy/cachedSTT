# Nemotron Speech ASR 0.6B - 모델 네트워크 구조

## 전체 파이프라인

```
 마이크 입력 (16kHz mono WAV)
      │
      ▼
┌─────────────────────────────────────────────────────────┐
│  1. Preprocessor (Mel Spectrogram 추출)                   │
│     16kHz 오디오 → 80차원 Mel Spectrogram                  │
│     25ms 윈도우, 10ms 스트라이드                            │
│     ★ 언어 무관 (물리적 음향 특성만 추출)                     │
│     파라미터: 학습 없음 (고정 수학 연산)                      │
└────────────────────┬────────────────────────────────────┘
                     │  [Time × 80] 텐서
                     ▼
┌─────────────────────────────────────────────────────────┐
│  2. Convolutional Subsampling (8x 다운샘플링)              │
│     Depthwise-Separable Convolution                      │
│     시간축을 1/8로 압축 → 연산 효율 극대화                    │
│     80차원 → 1024차원으로 확장                              │
│     ★ 언어 무관 (시간축 압축만 수행)                         │
└────────────────────┬────────────────────────────────────┘
                     │  [Time/8 × 1024] 텐서
                     ▼
┌─────────────────────────────────────────────────────────┐
│                                                          │
│  3. FastConformer Encoder (24층)                          │
│                                                          │
│  ┌────────────────────────────────────────────────────┐  │
│  │  하위 레이어 (Layer 0 ~ 11)                          │  │
│  │                                                     │  │
│  │  각 레이어 구조:                                      │  │
│  │  ┌───────────────────────────────────────────────┐  │  │
│  │  │                                               │  │  │
│  │  │  Input ──→ [Feed-Forward #1] ──→              │  │  │
│  │  │         ──→ [Self-Attention]  ──→              │  │  │
│  │  │         ──→ [Convolution]     ──→              │  │  │
│  │  │         ──→ [Feed-Forward #2] ──→ Output       │  │  │
│  │  │                                               │  │  │
│  │  │  ★ Self-Attention: 문맥 내 프레임 간 관계 학습     │  │  │
│  │  │  ★ Convolution: 지역적 음향 패턴 포착             │  │  │
│  │  │  ★ Feed-Forward: 비선형 변환                     │  │  │
│  │  │                                               │  │  │
│  │  └───────────────────────────────────────────────┘  │  │
│  │                                                     │  │
│  │  학습 내용:                                           │  │
│  │    Layer 0~3:  에너지 윤곽, 기본 주파수 패턴            │  │
│  │    Layer 4~7:  음소 경계, 모음/자음 구분               │  │
│  │    Layer 8~11: 음절 구조, 화자 정규화                  │  │
│  │                                                     │  │
│  │  ★ 이 부분은 모든 언어에 공통 → 영어 가중치 재사용      │  │
│  │  ★ Stage 1, 2에서 FROZEN (고정)                      │  │
│  └────────────────────────┬───────────────────────────┘  │
│                           │                              │
│  ┌────────────────────────▼───────────────────────────┐  │
│  │  상위 레이어 (Layer 12 ~ 23)                         │  │
│  │                                                     │  │
│  │  (동일한 레이어 구조 × 12층)                           │  │
│  │                                                     │  │
│  │  학습 내용:                                           │  │
│  │    Layer 12~15: 서브워드 단위 음향-텍스트 매핑          │  │
│  │    Layer 16~19: 문맥 의존적 음향 해석                  │  │
│  │    Layer 20~23: 고수준 언어 모델링                     │  │
│  │                                                     │  │
│  │  ★ 이 부분은 언어에 종속 → 한국어로 재학습 필요         │  │
│  │  ★ Stage 1에서 FROZEN, Stage 2에서 TRAINABLE         │  │
│  └────────────────────────┬───────────────────────────┘  │
│                           │                              │
│  ┌────────────────────────▼───────────────────────────┐  │
│  │  Layer Norm (최종 정규화)                              │  │
│  └────────────────────────┬───────────────────────────┘  │
│                           │                              │
└───────────────────────────┼──────────────────────────────┘
                            │  Encoder 출력: [Time/8 × 1024]
                            │
          ┌─────────────────┴─────────────────┐
          │                                   │
          ▼                                   ▼
┌──────────────────────┐    ┌──────────────────────────────┐
│ 4. Prediction Network │    │  (Encoder 출력을 직접 사용)     │
│    (RNN-T Decoder)    │    │                              │
│                       │    │                              │
│  이전 출력 토큰 →       │    │                              │
│  Embedding (1024차원)  │    │                              │
│       │               │    │                              │
│       ▼               │    │                              │
│  LSTM (2층, 640차원)   │    │                              │
│       │               │    │                              │
│       ▼               │    │                              │
│  Projection (640차원)  │    │                              │
│                       │    │                              │
│  ★ "다음에 어떤 토큰이  │    │                              │
│    올지" 언어 모델 역할  │    │                              │
│  ★ 한국어로 새로 학습    │    │                              │
└──────────┬────────────┘    └──────────────┬───────────────┘
           │                                │
           │  [640차원]                      │  [1024차원]
           │                                │
           ▼                                ▼
┌─────────────────────────────────────────────────────────┐
│  5. Joint Network (결합 네트워크)                          │
│                                                          │
│  Encoder 출력 (1024) ──→ [Linear] ──→ (640)              │
│                                          │               │
│  Prediction 출력 (640) ─────────────────→ +              │
│                                          │               │
│                                     [ReLU + Linear]      │
│                                          │               │
│                                    [Vocab Projection]    │
│                                          │               │
│                                     (4097차원)            │
│                                    4096 토큰 + blank      │
│                                                          │
│  ★ 음향 정보(Encoder)와 언어 정보(Prediction)를 결합        │
│  ★ 매 시간 스텝 × 매 토큰 위치에서 확률 계산                 │
│  ★ 한국어로 새로 학습                                      │
└────────────────────┬────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────┐
│  6. RNN-T Decoding (탐색)                                 │
│                                                          │
│  매 시간 스텝에서:                                         │
│    → blank (∅) 출력 = "아직 모르겠다" → 다음 프레임으로      │
│    → 토큰 출력 = "이 글자다!" → 텍스트에 추가               │
│                                                          │
│  예시:                                                    │
│  시간 →  t1    t2    t3    t4    t5    t6    t7           │
│  출력 →  ∅     "안"   ∅    "녕"  "하"  ∅    "세"          │
│  텍스트→              "안"       "안녕하"     "안녕하세"     │
│                                                          │
│  ★ Greedy decoding: 가장 높은 확률의 토큰 선택              │
│  ★ FastEmit: blank 대신 실제 토큰을 빨리 출력하도록 유도     │
└────────────────────┬────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────┐
│  7. Tokenizer (SentencePiece Unigram, vocab=4096)        │
│                                                          │
│  토큰 ID → 한국어 텍스트 변환                               │
│                                                          │
│  예시:                                                    │
│  [234, 1502, 89, 3201] → "안녕하세요 팀벨입니다"            │
│                                                          │
│  특수 토큰:                                                │
│    네, 아, 음, 어, 그 → 필러워드 (턴 디텍션용)               │
│    <EOU> → 발화 종료 마커 (미래 확장용)                      │
│    byte_fallback → 미등록 문자도 처리 가능                   │
└────────────────────┬────────────────────────────────────┘
                     │
                     ▼
              "안녕하세요 팀벨입니다"
```

## Cache-Aware Streaming 동작 방식

```
일반 ASR (오프라인):
  전체 오디오를 한 번에 처리 → 전사 결과

Buffered Streaming (기존 방식):
  오디오를 겹쳐가며 반복 처리 → 느리고 중복 연산 많음

Cache-Aware Streaming (우리 모델):
  각 chunk를 딱 한 번만 처리, 이전 정보는 cache에서 가져옴

┌─────────────────────────────────────────────────────────┐
│  시간 →                                                  │
│                                                          │
│  오디오:  [chunk1] [chunk2] [chunk3] [chunk4] ...         │
│           160ms    160ms    160ms    160ms                │
│                                                          │
│  처리:                                                    │
│                                                          │
│  chunk1 → Encoder 처리 → cache₁ 저장                      │
│                            │                             │
│  chunk2 → Encoder 처리 ←── cache₁ 재사용 → cache₂ 저장     │
│                                             │            │
│  chunk3 → Encoder 처리 ←────────────── cache₂ → cache₃    │
│                                                          │
│  ★ 핵심: 새 chunk만 처리, 이전 chunk는 cache에서 참조       │
│  ★ Self-Attention의 context를 cache로 유지                 │
│  ★ Convolution의 receptive field도 cache로 유지            │
│                                                          │
│  Multi-Lookahead (추론 시 레이턴시 선택):                   │
│    [70,13] → 1040ms 지연, 높은 정확도                      │
│    [70, 6] →  480ms 지연, 균형                             │
│    [70, 1] →   80ms 지연, 빠름                             │
│    [70, 0] →    0ms 지연, 가장 빠름 (정확도↓)               │
│                                                          │
│  ★ 학습 시 4가지를 랜덤으로 섞어 학습                        │
│  ★ 추론 시 용도에 맞게 선택 (재학습 불필요!)                  │
└─────────────────────────────────────────────────────────┘
```

```
          ┌─────────────────────┐

          └──────────┬──────────┘
                     │
          ┌──────────▼──────────┐
          │    Subsampling      │  항상 고정
          └──────────┬──────────┘
                     │
  ┌──────────────────▼──────────────────┐
  │  Encoder Layer 0~11 (하위)           │
  │                                      │
  │  Stage 1: ❄️ FROZEN                  │
  │  Stage 2: ❄️ FROZEN                  │
  │  Stage 3: 🔥 TRAINABLE              │
  └──────────────────┬──────────────────┘
                     │
  ┌──────────────────▼──────────────────┐
  │  Encoder Layer 12~23 (상위)          │
  │                                      │
  │  Stage 1: ❄️ FROZEN                  │
  │  Stage 2: 🔥 TRAINABLE  ← 현재 여기  │
  │  Stage 3: 🔥 TRAINABLE              │
  └──────────────────┬──────────────────┘
                     │
          ┌──────────▼──────────┐
          │  Prediction (LSTM)  │
          │  + Joint Network    │
          │                     │

          ┌──────────▼──────────┐

  ┌──────────────────────────────────────────────┐
```

## RNN-T vs CTC vs AED 비교

```
RNN-T (우리 모델 - Transducer):
  ┌─────────┐     ┌───────────┐
  │ Encoder │────→│   Joint   │────→ 토큰
  └─────────┘     └─────┬─────┘
                        │
                  ┌─────▼─────┐
                  │Prediction │←── 이전 토큰
                  └───────────┘
  ★ 스트리밍 가능, 이전 출력을 참고 (autoregressive)
  ★ 콜봇에 최적

CTC (Connectionist Temporal Classification):
  ┌─────────┐     ┌───────────┐
  │ Encoder │────→│  Linear   │────→ 토큰
  └─────────┘     └───────────┘
  ★ 스트리밍 가능, 단순하지만 이전 출력 참조 불가
  ★ 독립적 프레임 판단 → 문맥 부족

AED (Attention Encoder-Decoder, Canary 등):
  ┌─────────┐     ┌───────────────────┐
  │ Encoder │────→│ Transformer Decoder│────→ 토큰
  └─────────┘     │  (Cross-Attention) │
                  └───────────────────┘
  ★ 전체 입력을 봐야 함 → 스트리밍 불가
  ★ 오프라인 정확도 최고
```
