# 한국어 ASR 학습 데이터 준비 가이드

상용 한국어 음성인식 모델 학습을 위한 데이터 준비 절차.

## 1. 개요

### 데이터 구성

| 구분 | 원본 SR | 시간 | 용도 |
|------|---------|------|------|
| 16kHz 일반 한국어 | 16kHz | ~5,000시간 | 범용 한국어 음향 학습 |
| 8kHz 전화 음성 | 8kHz | ~8,000시간 | 전화 도메인 적응 |
| **합계** | | **~13,000시간** | |

### 처리 파이프라인 요약

```
원본 오디오 + 전사 텍스트
  → ① 텍스트 정규화: ITN 적용 (숫자·날짜·단위를 written form으로)
  → ② 텍스트 정규화: 구두점(. ?) 추가
  → ③ 16kHz 리샘플링 (8kHz 데이터만)
  → ④ 품질 필터링 (길이, 채널, SR, 텍스트 검증)
  → ⑤ Train/Val 분할 (95:5, 화자 단위)
  → ⑥ Speed Perturbation (Train만, 0.9x/1.0x/1.1x) → Train ~37,000시간
  → ⑦ FLAC 압축
  → ⑧ NeMo Tarred Dataset 생성 (tar 샤드)
  → ⑨ 클라우드 업로드
```

> **중요**: 텍스트 정규화(①②)를 오디오 처리(③~)보다 **먼저** 수행합니다.
> 모델이 최종 사용자에게 보여줄 형태 그대로를 학습해야 합니다.
>
> **중요**: Train/Val 분할(⑤)을 Speed Perturbation(⑥) **전에** 수행합니다.
> Validation에는 원본(1.0x)만 사용하여 공정한 평가를 보장합니다.

### 최종 산출물

## 2. 오디오 요구사항

### 필수 포맷

| 항목 | 값 | 비고 |
|------|-----|------|
| Sample Rate | **16,000 Hz** | 8kHz 데이터는 반드시 업샘플링 |
| Channels | **모노 (1ch)** | 스테레오는 좌채널 추출 또는 다운믹스 |
| Bit Depth | 16-bit | |
| 형식 | WAV (중간), FLAC (최종) | |
| 최소 길이 | **0.5초** | 이하는 제외 |
| 최대 길이 | **20.0초** | 초과 시 분할 필요 |

### 8kHz → 16kHz 리샘플링

```bash
# SOX (권장 — 고품질 리샘플링)
sox input_8k.wav -r 16000 output_16k.wav

# ffmpeg
ffmpeg -i input_8k.wav -ar 16000 -ac 1 output_16k.wav

# 일괄 변환 예시 (디렉토리 단위)
find /data/8k_raw/ -name "*.wav" | parallel -j $(nproc) \
    'sox {} -r 16000 /data/16k_resampled/$(basename {})'
```

> **주의**: 8kHz → 16kHz 업샘플링해도 4kHz 이상 주파수 정보는 생성되지 않음 (나이퀴스트 한계). 이것은 정상이며, 모델이 학습 과정에서 적응합니다.

### 스테레오 → 모노 변환

```bash
# 좌채널만 추출
sox input_stereo.wav output_mono.wav remix 1

# 또는 다운믹스 (좌+우 평균)
ffmpeg -i input_stereo.wav -ac 1 output_mono.wav
```

## 3. 텍스트 정규화 (ITN + 구두점)

모델이 출력하는 텍스트가 **최종 사용자에게 보여지는 형태**입니다.
따라서 학습 텍스트를 사용자가 기대하는 written form으로 정규화해야 합니다.

### 3-1. ITN (Inverse Text Normalization) 적용

ITN은 음성으로 발화된 형태(spoken form)를 문자 표기(written form)로 변환하는 것입니다.

**변환 대상 및 예시:**

| 카테고리 | Spoken Form (X) | Written Form (O) |
|----------|----------------|-----------------|
| 숫자 | 삼백이십오 | **325** |
| 금액 | 만 이천원 | **12,000원** |
| 날짜 | 이천이십육년 이월 구일 | **2026년 2월 9일** |
| 시간 | 오후 세시 삼십분 | **오후 3시 30분** |
| 전화번호 | 공일공 이삼사오 육칠팔구 | **010-2345-6789** |
| 퍼센트 | 삼십이점오퍼센트 | **32.5%** |
| 영어 약어 | 에스에스알아이 | **SSRI** |
| 주소 | 서울시 강남구 삼성동 백이십삼번지 | **서울시 강남구 삼성동 123번지** |
| 단위 | 백이십킬로그램 | **120kg** |
| 순서 | 첫번째, 두번째 | **1번째, 2번째** (또는 첫번째 유지) |

**ITN 적용 기준:**

1. **숫자**: 모든 숫자를 아라비아 숫자로 표기
   - "삼십만원" → "300,000원" 또는 "30만원"
   - 금액에 천 단위 콤마 포함 권장
2. **날짜/시간**: 숫자 + 단위 형태
   - "이월" → "2월", "세시" → "3시"
3. **전화번호**: 하이픈(-) 구분
   - "공일공" → "010", 3-4-4 또는 2-3-4 형식
4. **영어/약어**: 대문자 영문 표기
   - "에이아이" → "AI", "에스엔에스" → "SNS"
5. **단위**: 숫자 + 단위 기호
   - "킬로미터" → "km", "퍼센트" → "%"

**주의사항:**
- 이미 written form인 텍스트는 변환하지 않음
- 문맥에 따라 변환이 달라질 수 있음 (예: "하나" → 문맥에 따라 "1" 또는 "하나")
- 불확실한 경우 원본 유지가 안전

**ITN 도구/라이브러리:**

| 도구 | 특징 |
|------|------|
| NeMo Text Processing (`nemo_text_processing`) | NVIDIA 공식, WFST 기반, 한국어 일부 지원 |
| 자체 규칙 기반 | 정규식으로 패턴 매칭 (정확도 높지만 개발 비용) |
| LLM 기반 | GPT/Claude로 일괄 변환 (대량 처리 시 비용 고려) |

```bash
# NeMo Text Processing 사용 예시 (설치 필요)
pip install nemo_text_processing

python -c "
from nemo_text_processing.inverse_text_normalization.inverse_normalize import InverseNormalizer
itn = InverseNormalizer(lang='ko')
print(itn.inverse_normalize('삼백이십오'))  # → 325
"
```

### 3-2. 구두점(. ?) 추가

모델이 문장 끝에 구두점을 출력해야 문장 경계를 자동으로 감지할 수 있습니다.

**규칙:**

| 구두점 | 사용 위치 |
|--------|----------|
| **.** (마침표) | 평서문, 명령문, 청유문 끝 |
| **?** (물음표) | 의문문 끝 |

**적용 방법:**

1. 이미 구두점이 있는 텍스트 → 그대로 유지
2. 구두점이 없는 텍스트 → 한국어 종결어미 패턴으로 자동 추가

```
[구두점 없음] 오늘 날씨가 좋습니다
[마침표 추가] 오늘 날씨가 좋습니다.

[구두점 없음] 내일 비가 올까요
[물음표 추가] 내일 비가 올까요?

[이미 있음]   감사합니다. 이상입니다.  → 그대로
```

**주요 한국어 종결어미 패턴:**

마침표(.) 패턴:
- 격식체: ~합니다, ~입니다, ~겠습니다, ~습니다
- 해요체: ~해요, ~예요, ~어요, ~네요, ~거든요
- 반말: ~했어, ~봤어, ~거든, ~잖아
- 평서형: ~한다, ~된다, ~있다, ~없다
- 의지: ~할게, ~볼게
- 명령: ~하세요, ~주세요, ~바랍니다

물음표(?) 패턴:
- ~합니까, ~습니까, ~까요, ~나요, ~가요
- ~할까요, ~있나요, ~없나요, ~인가요

**적용 스크립트:**
```bash
python scripts/add_punctuation_to_manifest.py \
    --input manifest_itn.json \
    --output manifest_itn_punct.json
```

## 4. 품질 필터링

처리 전에 반드시 다음 항목을 검증하고 부적합 데이터를 제외합니다.

### 제외 기준

| 검사 항목 | 제외 조건 | 이유 |
|----------|----------|------|
| 길이 | < 0.5초 또는 > 20.0초 | 너무 짧으면 학습 불가, 길면 GPU OOM |
| 채널 수 | 스테레오(2ch) 그대로 | 반드시 모노로 변환 후 포함 |
| Sample Rate | 16kHz가 아닌 경우 | 리샘플링 후 포함 |
| 무음 파일 | RMS < 0.001 (전체) | 빈 오디오 |
| 텍스트 비어있음 | text == "" | 레이블 없음 |
| 텍스트 깨짐 | 비한국어 문자 비율 > 30% | 인코딩 오류 |

### 검증 스크립트 예시

```python
import soundfile as sf
import json
import numpy as np

def validate_audio(filepath):
    """오디오 파일 검증. 문제 있으면 이유 문자열 반환, 정상이면 None."""
    try:
        info = sf.info(filepath)
    except Exception as e:
        return f"읽기 실패: {e}"

    if info.channels != 1:
        return f"모노 아님: {info.channels}ch"
    if info.samplerate != 16000:
        return f"SR 불일치: {info.samplerate}"
    if info.duration < 0.5:
        return f"너무 짧음: {info.duration:.2f}s"
    if info.duration > 20.0:
        return f"너무 김: {info.duration:.2f}s"

    # 무음 검사
    audio, sr = sf.read(filepath)
    rms = np.sqrt(np.mean(audio ** 2))
    if rms < 0.001:
        return f"무음: RMS={rms:.6f}"

    return None  # 정상


def validate_manifest(manifest_path, output_path, report_path):
    """manifest 전체 검증 및 필터링."""
    total, passed, filtered = 0, 0, 0
    reasons = {}

    with open(manifest_path) as fin, open(output_path, 'w') as fout:
        for line in fin:
            total += 1
            d = json.loads(line)

            # 텍스트 검사
            if not d.get('text', '').strip():
                reasons['텍스트 비어있음'] = reasons.get('텍스트 비어있음', 0) + 1
                filtered += 1
                continue

            # 오디오 검사
            problem = validate_audio(d['audio_filepath'])
            if problem:
                reasons[problem.split(':')[0]] = reasons.get(problem.split(':')[0], 0) + 1
                filtered += 1
                continue

            passed += 1
            fout.write(line)

    # 리포트 생성
    with open(report_path, 'w') as f:
        f.write(f"총 샘플: {total:,}\n")
        f.write(f"통과: {passed:,} ({passed/total*100:.1f}%)\n")
        f.write(f"제외: {filtered:,} ({filtered/total*100:.1f}%)\n\n")
        f.write("제외 사유:\n")
        for reason, count in sorted(reasons.items(), key=lambda x: -x[1]):
            f.write(f"  {reason}: {count:,}\n")
```

## 5. Speed Perturbation

음성의 재생 속도를 변형하여 데이터 다양성을 3배로 증가시킵니다.

### 속도 배율

| 배율 | 효과 | 설명 |
|------|------|------|
| **0.9x** | 느리게 | 천천히 말하는 화자 시뮬레이션 |
| **1.0x** | 원본 | 그대로 유지 |
| **1.1x** | 빠르게 | 빨리 말하는 화자 시뮬레이션 |

### 변환 명령

```bash
# SOX로 speed perturbation (피치 유지하면서 속도 변경)
sox input.wav output_0.9.wav speed 0.9
sox input.wav output_1.1.wav speed 1.1

# 일괄 처리 예시
for speed in 0.9 1.0 1.1; do
    find /data/16k_clean/ -name "*.wav" | parallel -j $(nproc) \
        "sox {} /data/speed_perturbed/sp${speed}_{/.}.wav speed ${speed}"
done
```

### Manifest 생성 시 주의

- Speed perturbation 적용 후 **duration이 변경**됨
  - 0.9x → duration / 0.9 (길어짐)
  - 1.1x → duration / 1.1 (짧아짐)
- **반드시 변환된 파일의 실제 duration을 측정**하여 manifest에 기록

```python
import soundfile as sf

def get_actual_duration(filepath):
    info = sf.info(filepath)
    return info.duration
```

- 0.9x 적용 후 duration > 20.0초가 되는 샘플은 **제외**

## 6. FLAC 압축

WAV → FLAC 변환으로 파일 크기를 **약 60% 감소** (무손실).

```bash
# 단일 파일
ffmpeg -i input.wav -c:a flac output.flac

# 일괄 변환
find /data/speed_perturbed/ -name "*.wav" | parallel -j $(nproc) \
    'ffmpeg -y -i {} -c:a flac {.}.flac && rm {}'

# 또는 SOX
find /data/speed_perturbed/ -name "*.wav" | parallel -j $(nproc) \
    'sox {} {.}.flac && rm {}'
```

## 7. NeMo Tarred Dataset 생성 (필수)

수백만 개의 개별 파일 대신 **tar 샤드**로 묶어 학습 I/O를 최적화합니다.

### 왜 필요한가

| 방식 | 파일 수 | I/O 패턴 | GPU 활용률 |
|------|---------|---------|-----------|
| 개별 파일 | ~16,200,000 | 랜덤 읽기 | 50% 이하 |
| **Tar 샤드** | ~3,000개 | **순차 읽기** | **95%+** |

### 샤드 생성 규칙

| 항목 | 값 | 비고 |
|------|-----|------|
| 샤드 크기 | ~512MB | 너무 크면 셔플 저하, 너무 작으면 파일 수 증가 |
| 샤드 수 | GPU 수의 배수 | 4 GPU 사용 시 4의 배수 (예: 3000, 3004) |
| 파일 형식 | `.tar` (비압축) | 내부 오디오가 이미 FLAC 압축됨 |

### 생성 스크립트

```python
#!/usr/bin/env python3
"""
NeMo Tarred Dataset 생성기

Usage:
    python create_tarred_dataset.py \
        --manifest input_manifest.json \
        --output_dir /data/tarred_train \
        --num_shards 3000 \
        --shuffle
"""
import argparse
import json
import os
import random
import tarfile
from pathlib import Path

def create_tarred_dataset(manifest_path, output_dir, num_shards, shuffle=True):
    os.makedirs(output_dir, exist_ok=True)

    # 1. Manifest 로드
    entries = []
    with open(manifest_path) as f:
        for line in f:
            entries.append(json.loads(line))

    if shuffle:
        random.shuffle(entries)

    print(f"총 샘플: {len(entries):,}")
    print(f"샤드 수: {num_shards}")
    print(f"샤드당 샘플: ~{len(entries) // num_shards:,}")

    # 2. 샤드별로 분배
    shard_entries = [[] for _ in range(num_shards)]
    for i, entry in enumerate(entries):
        shard_entries[i % num_shards].append(entry)

    # 3. Tar 샤드 생성 + 새 manifest
    new_manifest_lines = []

    for shard_id in range(num_shards):
        tar_path = os.path.join(output_dir, f"audio_{shard_id}.tar")

        with tarfile.open(tar_path, "w") as tar:
            for entry in shard_entries[shard_id]:
                audio_path = entry["audio_filepath"]
                ext = Path(audio_path).suffix  # .flac or .wav
                # tar 안에서의 파일 이름
                arcname = Path(audio_path).name

                tar.add(audio_path, arcname=arcname)

                # manifest 업데이트: audio_filepath를 파일명만으로
                new_entry = {
                    "audio_filepath": arcname,
                    "duration": entry["duration"],
                    "text": entry["text"],
                    "shard_id": shard_id,
                }
                if "offset" in entry:
                    new_entry["offset"] = entry["offset"]
                new_manifest_lines.append(json.dumps(new_entry, ensure_ascii=False))

        if (shard_id + 1) % 100 == 0:
            print(f"  샤드 {shard_id + 1}/{num_shards} 완료")

    # 4. Manifest 저장
    manifest_out = os.path.join(output_dir, "manifest.json")
    with open(manifest_out, "w") as f:
        f.write("\n".join(new_manifest_lines) + "\n")

    print(f"\n완료!")
    print(f"  Tar 샤드: {output_dir}/audio_0.tar ~ audio_{num_shards-1}.tar")
    print(f"  Manifest: {manifest_out}")
    print(f"  총 샘플: {len(new_manifest_lines):,}")


if __name__ == "__main__":
    parser = argparse.ArgumentParser()
    parser.add_argument("--manifest", required=True)
    parser.add_argument("--output_dir", required=True)
    parser.add_argument("--num_shards", type=int, default=3000)
    parser.add_argument("--shuffle", action="store_true", default=True)
    parser.add_argument("--seed", type=int, default=42)
    args = parser.parse_args()

    random.seed(args.seed)
    create_tarred_dataset(args.manifest, args.output_dir, args.num_shards, args.shuffle)
```

### Manifest 형식 (tarred)

```jsonl
{"audio_filepath": "sp1.0_sample_001.flac", "duration": 8.5, "text": "안녕하세요.", "shard_id": 0}
{"audio_filepath": "sp0.9_sample_002.flac", "duration": 9.4, "text": "오늘 날씨가 좋습니다.", "shard_id": 0}
{"audio_filepath": "sp1.1_sample_003.flac", "duration": 7.7, "text": "감사합니다.", "shard_id": 1}
```

**필수 필드:**
- `audio_filepath`: tar 안에서의 파일 이름 (경로 아닌 파일명만)
- `duration`: 초 단위 길이 (float)
- `text`: 전사 텍스트 (한국어)
- `shard_id`: 해당 샘플이 들어있는 tar 샤드 번호 (int)

## 8. Train/Validation 분할

> **주의**: 분할은 Speed Perturbation 전에 수행합니다.
> 원본 데이터를 먼저 train/val로 나눈 뒤, train에만 speed perturbation을 적용합니다.
> Validation 데이터에는 **원본(1.0x)만** 사용합니다.

### 분할 비율

| 구분 | 비율 | 예상 시간 |
|------|------|----------|
| Train | **95%** | ~37,050시간 (speed perturb 후) |
| Validation | **5%** | ~1,950시간 |

### 분할 규칙

- **화자 단위 분할**: 같은 화자가 train과 val에 모두 포함되면 안 됨
- 화자 정보가 없으면 파일/세션 단위로 분할
- Validation 데이터에는 **Speed Perturbation 미적용** (원본 1.0x만)

## 10. 전수 검증 (학습 전 필수)

**학습 시작 전에 반드시 전수 검증을 실행합니다.**
검증을 건너뛰면 학습 도중 크래시가 발생할 수 있습니다. (실제로 스테레오 파일 혼입으로 학습 중단된 경험 있음)

### 검증 스크립트

```bash
# 스크립트 위치
scripts/validate_manifest.py
```

### 검증 항목

| 검사 | 조건 | 학습 중 영향 |
|------|------|------------|
| 파일 존재 | 경로에 파일 있는지 | FileNotFoundError 크래시 |
| 파일 읽기 | 깨진/잘린 파일 | 데이터 로더 크래시 |
| Sample Rate | ≠ 16,000Hz | mel 스펙트로그램 크기 불일치 |
| 채널 수 | ≠ 1 (모노) | 텐서 shape 불일치 크래시 |
| Duration 범위 | < 0.5s 또는 > 20.0s | OOM 또는 빈 텐서 |
| Duration 일치 | manifest vs 실제 차이 > 0.1s | 배치 패딩 비효율 |
| 무음 | RMS < 0.001 | 의미 없는 학습 |
| NaN/Inf | 오디오 값에 NaN/Inf | loss NaN → 학습 발산 |
| 텍스트 비어있음 | text == "" | 빈 레이블 에러 |

### 사용법

```bash
# 1. 검증만 (리포트 출력)
python scripts/validate_manifest.py \
    --manifest /data/manifest.json \
    --workers 8

# 2. 검증 + 문제 파일 자동 제거 → 클린 manifest 생성
python scripts/validate_manifest.py \
    --manifest /data/manifest.json \
    --fix --output /data/manifest_clean.json \
    --workers 8 \
    --report /data/validation_report.txt

# 3. Train + Val 둘 다 검증
python scripts/validate_manifest.py --manifest /data/train_manifest.json --fix --output /data/train_clean.json --workers 8
python scripts/validate_manifest.py --manifest /data/val_manifest.json --fix --output /data/val_clean.json --workers 8
```

### 검증 통과 기준

| 기준 | 값 |
|------|-----|
| 정상 비율 | **≥ 98%** |

## 11. 최종 체크리스트

작업 완료 후 다음 항목을 확인합니다.

### 오디오 검증

- [ ] 모든 파일이 16kHz, 모노, 16-bit
- [ ] 0.5초 미만 / 20.0초 초과 파일 제외됨
- [ ] 무음/깨진 파일 제외됨
- [ ] 8kHz 데이터가 16kHz로 리샘플링됨
- [ ] Speed perturbation 3가지 속도 (0.9, 1.0, 1.1) 모두 생성됨

### 텍스트 검증

- [ ] ITN 적용됨 (숫자·날짜·금액·전화번호가 written form)
- [ ] 구두점 추가 적용됨 (전체의 약 60% 이상에 `.` 또는 `?` 포함)
- [ ] 빈 텍스트 샘플 제외됨
- [ ] 인코딩 정상 (UTF-8 한국어)
- [ ] 랜덤 100개 샘플링하여 텍스트 품질 육안 확인

### 전수 검증 (필수)

- [ ] `validate_manifest.py` 실행 완료
- [ ] SR 불일치 0건
- [ ] 채널 불일치 0건
- [ ] 파일 없음 0건
- [ ] NaN/Inf 0건
- [ ] 정상 비율 ≥ 98%
- [ ] 검증 리포트 파일 보관 (`validation_report.txt`)

### Tarred Dataset 검증

- [ ] 샤드 수가 4의 배수 (GPU 4대 사용 예정)
- [ ] 각 샤드 크기 ~512MB
- [ ] manifest.json의 shard_id와 실제 tar 파일 일치
- [ ] manifest 총 라인 수 = 전체 샘플 수
- [ ] 샘플 tar 파일을 열어 오디오 재생 확인
